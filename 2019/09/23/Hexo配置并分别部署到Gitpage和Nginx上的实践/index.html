    Hexo配置并分别部署到Gitpage和Nginx上的实践

    建站 hexo nginx gitpage github 建站
    创建时间:2019-09-23 17:46
    字数:1k 阅读:

    2. 2. Hexo搭建
    3. 3. Hexo部署到GitPage
    4. 4. Hexo部署到Nginx
    摘要
    本篇文章主要介绍了以下内容:
    • Hexo的搭建和基本配置
    • 如何将Hexo博客部署到gitpage上
    • 如何将Hexo博客部署到第三方服务器（Nginx）上

    1. Hexo简介

    Hexo是一个基于NodeJs的博客制作工具，本身是可以当做博客服务器来使用，但是为了更好的性能，一般会将Hexo产生的静态博客页面(HTML)部署到专业的服务器上使用。
    所以Hexo更是一个静态页面生生成、部署工具

    2. Hexo搭建

    1. Hexo 准备
      Hexo底层基于NodeJS和Git, 所以请确保它们已经就位。如果没有，请参考: Hexo官方文档

    2. Hexo 安装

      $ npm install -g hexo-cli

    3. Hexo 创建

      $ hexo init <folder>  # folder是博客系统的名称
$ cd <folder>
$ npm install

    4. Hexo 目录结构

      ├── _config.yml #除主题中的 _comfig.xml外，最常用的一个配置文件
├── package.json #依赖包
├── scaffolds    #模板文件
├── source
|   ├── _drafts
|   └── _posts   #具体博客页面
└── themes       #主题目录

    5. Hexo 创建博客页面

      $ hexo new post <title>  # title就是博客页的标题

    6. 编辑Hexo博客页面
      基于上一步中创建的博客页面，可以使用第三方markdown编辑工具继续编辑博客页面，直至完成页面内容。markdown语法如下所示:

      >## Hexo搭建
>
>1. Hexo 准备
>Hexo底层基于NodeJS和Git, 所以请确保它们已经就位。如果没有，请参考: [Hexo官方文档](https://hexo.io/zh-cn/docs/)
>
>2. Hexo 安装
>```bash
>$ npm install -g hexo-cli
>```

      获取更多markdown语法

    7. 生成静态博客页面

      $ hexo clean
$ hexo generate

      现在已经生成了一个piblic文件夹，文件夹包含着所有包装好的博客内容页面。而后面的部署操作的实际就是操作这个文件夹。

    3. Hexo部署到GitPage

    1. GitPage是什么
      Gitpage是由GitHub托管的项目空间，免费，免运维。所以成为很多人的项目空间首选。

    2. GitHub 端的设置

      1. 项目的名称一定跟github账户名一致，并且项目名称后面一定加上’.github.io’.

      2. 项目设置中，关于gitpage的设置：

    3. Hexo 中的配置:

      deploy:
type: git
repo: git@github.com:xxxxx/xxxxx.github.io.git 
branch: master

      其中 xxxxx 是你的github账户名称。

    4 hexo端 和 github 端都设置好之后，就可以把’hexo generate’ 生成的public文件夹，部署到 gitpage了，通过以下命令:

    $ hexo deploy

    注意:hexo3.0以上，需要配合安装以下插件，才能上传成功:

    $ npm install hexo-deployer-git --save

    上传成功以后,就能在github项目下，看到上传的内容。
    并且，通过访问: https://xxxxx.github.io 就能够访问你的博客了！！
    其中 xxxxx 是你的github账户名称。

    4. Hexo部署到Nginx

    1. Nginx简介
      Nginx作为当今非常主流的web服务器，同时具有代理和反向代理等功能。用户覆盖率极高。今天我们就把Hero生成的博客静态HTML文件，部署到Nginx上。

    2. Nginx安装配置不做介绍，自己去掌握。

    3. 部署的实质就是将’hexo generate’ 生成的public文件夹，作为一个nginx的server去运行。

      1. 在Nginx中新增一个server,设置hexo的根目录为:’/opt/xxxxx/keepup/‘:

        # server hexo
server {
  listen       4000 ssl http2;
  listen       [::]:4000 ssl http2;
  server_name  www.whsasf.com localhost;
charset UTF-8;
  root        /opt/xxxxx/keepup/;

  # It is *strongly* recommended to generate unique DH parameters
  # Generate them with: openssl dhparam -out /etc/pki/nginx/dhparams.pem 2048
  #ssl_dhparam "/etc/pki/nginx/dhparams.pem";
  ssl_certificate "/etc/letsencrypt/live/whsasf.com/fullchain.pem";
ssl_certificate_key "/etc/letsencrypt/live/whsasf.com/privkey.pem";
ssl_session_cache shared:SSL:1m;
  ssl_session_timeout  10m;
  ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
  ssl_ciphers HIGH:SEED:!aNULL:!eNULL:!EXPORT:!DES:!RC4:!MD5:!PSK:!RSAPSK:!aDH:!aECDH:!EDH-DSS-DES-CBC3-SHA:!KRB5-DES-CBC3-SHA:!SRP;
  ssl_prefer_server_ciphers on;
  # Load configuration files for the default server block.
  include /etc/nginx/default.d/*.conf;

  location /{
  }
# static
location /static/{
  autoindex on;
}
# favicon
location = /favicon.ico {
  log_not_found off;
  access_log off;
}
  error_page 404 /40x.html;
      location = /40x.html {
  root   /opt/xxxxx/gt_web/error_page;
  }

  error_page 500 502 503 504 /50x.html;
      location = /50x.html {
  root /opt/xxxxx/gt_web/error_page;
  }
}

      2. 将public文件夹拷贝到nginx server指定的路径’/opt/xxxxx/keepup/‘。如果在远程机器中部署，可以使用scp等工具。

      3. 然后，重新启动Nginx服务, 就可以在浏览器中访问 hexo博客了!
        示例中的访问地址应为: https://www.whsasf.com:4000
